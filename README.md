## 📌 О проекте
Данный репозиторий содержит backend-часть MVP системы управления складом, спроектированной для автоматизации процессов 3PL-оператора. Проект разработан в рамках комплексного аудита и реинжиниринга бизнес-процессов складской логистики.

**Роль в проекте:** Системный аналитик / Backend-разработчик.

## ⚙️ Архитектура и Стек
Проект построен на базе классической клиент-серверной архитектуры с акцентом на отказоустойчивость и строгую консистентность данных.

* **Backend:** Python, Django, Django REST Framework (DRF)
* **СУБД:** PostgreSQL (спроектирована реляционная модель в 3НФ)
* **Интеграции:** REST API для взаимодействия с мобильными терминалами сбора данных (ТСД) на React Native.
* **Бизнес-логика:** Реализован алгоритм FIFO, строгая ролевая модель доступа (RBAC), система статусов ячеек в реальном времени.

## 📊 Модель данных (ER)
База данных спроектирована с учетом предотвращения аномалий при обновлении и удалении. Основные сущности:
1. `Order` — агрегация клиентских заказов.
2. `SKU` — товарные номенклатуры со штрихкодами.
3. `Storage_Bin` — топология склада (ячейки, стеллажи, зоны).

## 🚀 Основные эндпоинты API (Примеры)
* `POST /api/v1/scan/` — обработка сканирования штрихкода с мобильного ТСД.
* `GET /api/v1/inventory/` — получение текущих остатков в реальном времени.
* `PATCH /api/v1/orders/{id}/status/` — обновление статуса сборки.
